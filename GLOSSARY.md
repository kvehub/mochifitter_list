# もちふぃったープロファイル一覧 - 用語集

## プロジェクト概要

### 目的
VRChat用アバター向けの「もちふぃった～」プロファイル情報を収集・管理し、ユーザーが検索・閲覧できる静的Webサイトと管理ツール群を提供する。

### プロジェクトの役割
**もちふぃった～**は衣装変換ツールであり、各アバター用の**プロファイル（変換用ファイル群）**が必要となる。
本プロジェクトは、どのアバター用のプロファイルが、どこで配布されているかという**プロファイル情報**を一元管理するデータベースとWebサイトである。

### 構成
- **Webサイト**: プロファイル情報の検索・閲覧（index.html, lite.html）
- **管理ツール**: プロファイル情報の登録・編集（profile_editor.py, profile_editor_lower.py など）
- **データベース**: profiles.json（プロファイル情報のメタデータを格納）

## 用語一覧

### もちふぃった～関連

#### もちふぃった～
VRChat用の衣装を非対応アバター間で変換するツール。
衣装変換には変換元・変換先アバターの**プロファイル**（変換用ファイル群）と、変換先アバター本体が必要。

**注意点:**
- アニメーションや複雑なギミックは動作しなくなる可能性がある
- 変換には通常5-15分程度、ポリゴン数が多い場合は2時間以上かかることもある

**公式リンク:**
- [使用許諾契約書](https://docs.google.com/document/d/1IxuPIQCUL_ANMVFl4xVegD4Eb80MGhMqous0sf9tAAQ/edit?usp=sharing)
- [Q&A（よくある質問）](https://docs.google.com/document/d/1hwKzkDWmcrrz96oj1X_Agb5XWCtdqV9i5FfI5AMKJZc)
- [Discordサーバー](https://discord.gg/7FTYwxbpJC)

#### プロファイル
**もちふぃった～**で衣装変換を行うために必要な、アバターごとの変換情報が含まれたファイル群。
多くの場合、アバターを販売しているショップ（Boothなど）で配布されている。

**配置場所:**
- Unity: `Assets/OutfitRetargetingSystem/Editor` 直下

**種類:**
- **順変換プロファイル**: 他のアバターの衣装を自分のアバターへ合わせる
- **逆変換プロファイル**: 自分のアバターの衣装を他のアバターへ合わせる

**同梱プロファイル（もちふぃった～本体に含まれるもの）:**
- しなの（ポンデロニウム研究所）
- 桔梗（ポンデロニウム研究所）
- マヌカ（STUDIO JINGO）
- ベリル（NINE GATES）

#### 順方向変換
他のアバターの衣装をあるアバターへ合わせる方向の変換。
変換先アバターのプロファイルが順方向に対応している必要がある。

**例:**
- アバターAの衣装 → アバターBへ変換（アバターBの順方向プロファイルを使用）

#### 逆方向変換
あるアバターの衣装を他のアバターへ合わせる方向の変換。
変換元アバターのプロファイルが逆方向に対応している必要がある。

**例:**
- アバターAの衣装 → アバターBへ変換（アバターAの逆方向プロファイルを使用）

**必要なもの:**
- アバターAの逆方向プロファイル
- アバターBの順方向プロファイル
- アバターB本体

#### 中間素体（Template）
衣装変換の際に経由する標準的な体型。
変換は「変換元アバター → 中間素体 → 変換先アバター」という2段階で行われる。

#### 共通素体（bodyBase）
複数のアバターで共通して使用される基本体型。
共通素体を使用することで、他のアバター用の衣装も着用できるようになる。

**入力内容:**
- 基本的には共通素体プロジェクトの名前を入力
- 名前がない場合は、同じ素体を使用している別のアバター名を入力

**例:**
- 「Kikyo Base」「Manuka Base」など

**注意:**
- 「桔梗系」「マヌカ系」のような呼び方はしない

### プロジェクト関連

#### プロファイル情報
アバター名、作者、配布場所、価格などのメタデータ。
**profiles.json**に格納され、Webサイトで検索・閲覧できる。

**プロファイルとの違い:**
- **プロファイル**: 実際の変換用ファイル群（もちふぃった～で使用）
- **プロファイル情報**: プロファイルについてのメタデータ（本プロジェクトで管理）

#### レコード
**profiles.json**内の各プロファイル情報エントリ（1アバター分のデータ）。

**構成:**
- id, registeredDate, updatedDate
- avatarName, avatarNameUrl, avatarAuthor, avatarAuthorUrl
- bodyBase（共通素体）
- profileAuthor, profileAuthorUrl
- downloadMethod, downloadLocation
- pricing, price, avatarPrice
- official, forwardSupport, reverseSupport
- notes（備考）
- など

---

### データ構造

#### profiles.json
プロファイル情報を格納するJSONデータベース。

**構造:**
```json
{
  "lastUpdated": "2025-12-01 00:23:41 JST",
  "profiles": [
    {
      "id": "001",
      "avatarName": "透羽",
      ...
    },
    ...
  ]
}
```

#### フィールド
レコード内の各項目（id、avatarName、pricingなど）。

**主要フィールド:**
- **id**: プロファイル情報の一意な識別子（3桁の数字）
- **avatarName**: アバター名
- **avatarAuthor**: アバター作者名
- **bodyBase**: 共通素体の名前
- **profileAuthor**: プロファイル作成者名
- **official**: 公式プロファイルかどうか
- **forwardSupport**: 順方向変換に対応しているか
- **reverseSupport**: 逆方向変換に対応しているか
- **pricing**: 価格区分（無料/単体有料/アバター同梱）
- **price**: プロファイル価格
- **avatarPrice**: アバター価格

### 配布・価格関連

#### 公式プロファイル
アバター作者が提供するプロファイル。
多くの場合、アバター本体と同じページで配布されている。

#### 非公式プロファイル
第三者が作成したプロファイル。
別のページで配布されることが多い。

#### 価格区分
プロファイルの配布形態。

**種類:**
- **無料**: プロファイルが無料で配布されている
- **単体有料**: プロファイルが単体で有料販売されている
- **アバター同梱**: アバター本体の購入に含まれている

#### 配布方法
プロファイルの配布サイト。

**主な配布方法:**
- **Booth**: BOOTH（ブース）で配布
- **GoogleDrive**: Googleドライブで配布
- **GitHub**: GitHubで配布
- **Discord**: Discordサーバーで配布

### 管理ツール

#### profile_editor.py
プロファイル情報を編集するGUIツール（フル機能版）。

**特徴:**
- BeautifulSoup4に依存
- Boothページからの自動情報取得機能
- URL調査パネル
- ID振り直し機能
- 検索機能

**依存ライブラリ:**
- tkinter, tkcalendar, Pillow, requests, beautifulsoup4

#### profile_editor_lower.py
プロファイル情報を編集するGUIツール（軽量版）。

**特徴:**
- 依存ライブラリが少ない
- 基本的な編集機能のみ

**依存ライブラリ:**
- tkinter, tkcalendar, Pillow, requests

#### その他のツール
- **booth_url_extractor.py**: Booth URLを抽出
- **diff_checker.py**: プロファイル情報の差分チェック
- **url_investigation.py**: URL調査ツール

## 登録作業の流れ

1. **事前準備**: profile_editor.py起動
2. **URL収集**: BOOTHで検索 → booth_url_extractor.py実行
3. **差分確認**: diff_checker.py実行 → 未登録アバター抽出
4. **URL調査**: url_investigation.pyで各URLを確認
5. **判定と登録**:
   - 非登録対象（衣装・テクスチャ等）→ ブロック登録
   - 非公式プロファイル → 対応アバター調査後、レコード追加
   - 公式プロファイル → レコード追加、自動入力
6. **情報入力**: アバター情報、価格、配布場所などを入力
7. **保存とプッシュ**: profiles.jsonを保存 → GitHubにプッシュ

詳細は[README.md](README.md)のフローチャートを参照。
